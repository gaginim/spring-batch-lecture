    - JPA 어댑터 구현
        - OR 매핑 설정
            - 도메인 모델 (POJO)에 OR매핑을 하는것은 도메인 기술을 오염시키는 것
            - 도메인 모델이 데이터 영역과 관련된 이유로 변경되어야 할 수도 있음 (단일 책임 원칙 위반)
            - 그러나 OR 매핑을 위한 별도의 영속성 모델을 만드는 것은 너무 많은 "보일러 플레이트 코드" 를 양산함. 매핑 구현에 상당한 노력이 필요함
            - [***] 단순하고 복잡하지 않은 유스케이스라면 도메인 모델에 OR매핑을 하는것이 좋음
            - 도메인 모델 vs 데이터 모델 매핑
